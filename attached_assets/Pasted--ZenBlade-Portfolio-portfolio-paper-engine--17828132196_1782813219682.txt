Продовжуємо ZenBlade Portfolio.

Потрібно додати тест portfolio paper engine, щоб перевірити сценарій:
scanner знаходить strong BUY setup → portfolio engine відкриває paper-position.

Реальні ордери НЕ робити. OKX API ключі НЕ використовувати.

1. Онови src/portfolioPaperEngine.js

Зміни runPortfolioOnce так, щоб вона приймала optional options:

export async function runPortfolioOnce(options = {}) {

Підтримай такі options:
- scanProvider
- statePath
- tradesPath

Логіка:
- якщо options.statePath передано — читати state звідти, інакше data/state.json
- якщо options.tradesPath передано — PaperEngine має отримати цей tradesPath
- якщо options.scanProvider передано — використовувати його замість scanPortfolio(config)

Тобто:

const results = options.scanProvider
  ? await options.scanProvider()
  : await scanPortfolio(config)

Коли створюєш PaperEngine для відкриття позиції, передай:

new PaperEngine(
  { ...config, symbol: best.symbol },
  {
    candlesProvider: async () => best.candles,
    statePath,
    tradesPath
  }
)

Коли моніториш існуючу позицію, теж передай statePath і tradesPath.

Звичайний npm run portfolio:start має працювати як раніше.
